1. 查看内存使用情况

1.1. 查看设备内存占用

1.1.1. /proc/meminfo

MemTotal：总的物理内存大小

MemFree：未使用的物理内存大小

MemAvailable：可使用的物理内存大小（除了MemFree的内存，还包括已经使用但可以回收的内存，比如Buffer/Cache等中的一部分）

1.1.2. free

total：总的物理内存大小

used：已使用物理内存大小

free：未使用的物理内存大小

shared：进程共享内存的内存大小

buff/cache：磁盘缓存的大小

available：可使用的物理内存的大小

1.2. 查看进程内存

1.2.1. top命令

使用top -H就会出现如下图所示的界面，也可以通过top -p pid查看指定进程内存占用情况（单位KB）

  • VIT(virtual memory)：进程申请的虚拟内存空间的大小，包括映射到物理内存的空间以及未映射的空间
  • RES(resident memory)：进程虚拟内存空间中已经映射到物理内存空间的大小
  • SHR(shared memory)：共享物理内存大小，比如动态库，不同进程中动态库的位置都指向相同的物理内存空间

1.2.2. proc/进程号/status

cat proc/进程号/status查看进程的内存使用具体情况

  • VmSize（virtual memory size）：虚拟内存大小（这里大小应该不是text/data/lib等加起来的总和）
  • VmHWM（virtual memory high water mark）：分配到的物理内存峰值
  • VmRSS（virtual memory resident set size)：正在使用的物理内存大小，是RssAnon(Rss Anoymous memory)、Rss File、RssShmem（Rss Shared Memory）的总和
  • VmData：数据段占用的虚拟内存大小（bss/initialized data/heap)
  • VmStk：堆栈占用的虚拟内存大小
  • VmExe：代码段占用的虚拟内存大小
  • VmLib：共享库占用的虚拟内存大小
  • VmPTE（virtual memory page table entries）：进程页表大小
  • VmPMD：进程二级页表大小

1.2.3. proc/进程号/statm

输入命令后显示的大小依次代表（页面为单位，目前看x86虚拟机上一张页表占用4KB大小）：

Size：和1.2.2.中的VmSize一致

Resident：和1.2.2.中的VmRss一致

Shared：和1.2.2.中的RssFile和RssShmem总和一致

text：代码段页数

lib：lib库页数，一直都是0，从linux 2.6后的版本就没用过了

data：data段和栈段的页数（看起来是VmData和VmStk的总和）

dt（dirty page）：表示已经修改的物理内存，一直都是0，从linux 2.6后的版本就没用过了

1.2.4. 查看进程内存映射

cat /proc/进程号/maps

2. 内存优化

2.1. 字节对齐

2.1.1. 为什么需要字节对齐

        对于struct结构体中，如果没有字节对齐，则可能会出现多次访问内存的情况。比如假设系统每次读取4字节数据，对于以下结构体，i分配1字节，j分配4字节，没有字节对齐的情况下，总体占用内存是5字节，如果要访问j，就需要取两次地址；使用字节对齐后，总体占用内存是5字节，i和j之间会自动进行填充，总体占用8字节。

struct test {

    char i;

    int j;

}

2.1.2. 字节对齐规则

  • 每个结构体成员相对结构体偏移的地址是该成员大小的整数倍
  • 结构体总体大小是最大成员的整数倍

2.1.3. 优化

从字节对齐的概念可以看到，典型的空间换时间。所以可以排查下结构体中有没有因为字节对齐而多分配的空间，能否调整位置或者删除不必要的成员进行优化（最好是经常使用的数据结构）

2.2. 位域

C语言提供了按位存取的功能。在对于某些只需要存取两种值（0和1）的数据（一般是用bool类型，但bool类型占用4字节），可以直接使用位域存取

struct test {

    unsigned int i: 1;

    unsigned int j: 1;

}

上面结构实际占用了4个字节，但i和j只占用了其中2位

2.3. 重复的数据结构

这里涉及到的就是业务中重复度的问题了，经常使用的数据尽量抽象出来当作基本数据结构。
